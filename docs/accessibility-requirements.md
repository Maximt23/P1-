# Accessibility Requirements

WearWise is an accessibility-first application. Key requirements include:

*   **Color Blindness Support**: All UI elements, color-coding, and information display must be perceivable and understandable by users with various forms of color blindness. Avoid relying solely on color to convey meaning.
*   **High Contrast**: Ensure sufficient contrast ratios for text and interactive elements.
*   **Large Text Support**: The app must respect system-wide large text settings.
*   **VoiceOver/Screen Reader Compatibility**: All interactive elements and important information must be properly labeled and accessible via screen readers.
*   **Intuitive Navigation**: Clear and consistent navigation patterns.
*   **Non-Judgmental Language**: Especially for the Quick Color Check and Tuck Logic features, language must be supportive and informative, never judgmental.
*   **Haptic Feedback**: Consider for key interactions (e.g., successful save, critical alerts).
*   **Adjustable Animation Speeds**: User preference for reducing motion.
