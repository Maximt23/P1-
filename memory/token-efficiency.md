# Token Efficiency Log

**Purpose:** To monitor token usage patterns, identify waste, and record better prompting and workflow patterns to ensure the startup stays lean, reduces costs, and optimizes execution speed and quality.

## Entries:

### Situation: (YYYY-MM-DD) [Brief description of the context or task]
-   **Waste observed:** Describe the specific inefficiency or excessive token usage (e.g., redundant prompts, overly verbose reasoning, unnecessary file creation, over-engineered workflows).
-   **Likely cause:** Why did this waste occur? (e.g., unclear instructions, lack of pattern reuse, agent drift, suboptimal prompt structure).
-   **Better pattern:** What is the more efficient or effective approach, prompt structure, or workflow that addresses the waste?
-   **Recommended future behavior:** Concrete guidance for agents to avoid this waste and adopt the better pattern in similar situations.
-   **Estimated impact:** Quantify or describe the expected savings in tokens, time, or improvement in quality/speed.
