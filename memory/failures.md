# Failures Log

**Purpose:** To record failed approaches, document what broke, explain why it failed, and capture what should be avoided to improve future execution quality and reduce repeated mistakes.

## Entries:

### Failure: (YYYY-MM-DD) [Brief, descriptive title of the failure]
-   **Context:** What was the situation or goal when this failure occurred?
-   **What was attempted:** Describe the approach, workflow, or code that led to the failure.
-   **Why it failed:** Explain the root cause(s) of the failure. What went wrong?
-   **Signals that it was failing:** What indicators were present that could have warned us?
-   **What to avoid next time:** Concrete actions or patterns to steer clear of in similar situations.
-   **Better alternative if known:** If a successful alternative was found or can be proposed, describe it.
